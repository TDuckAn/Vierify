# Vierify — Claude Design Brief

> **Audience:** Claude Design only. Do not edit this file; update requests go through Claude (planning role).
> **Scope:** Web (Next.js) + Mobile (Expo). Desktop is a web-wrapper and inherits the web design automatically.

## 1. Project in One Sentence

Vierify is a supply chain traceability platform for Vietnamese businesses — merchants scan and link product batches (B2B mobile app), consumers scan a QR code to see the farm-to-table origin story with blockchain proof (B2C website).

---

## 3. Design Direction

### Feel
Minimalist and structured — every element earns its space. The product must feel **professional, secure, and trustworthy** (blockchain proof, government-grade traceability) while remaining **approachable and not intimidating** for factory workers on mobile. Think: a premium B2B SaaS tool that a non-technical warehouse manager can use confidently on their first day.

### Personality
- Clean whitespace over decoration
- Data is the hero — surfaces information without clutter
- Confident UI copy (short, direct, in Vietnamese by default)
- Animations signal meaning (not decoration): loading states, scan success, blockchain confirmation

### References to draw from (combine selectively)
- Linear (clean data dashboards, tight typography)
- Vercel Dashboard (developer-grade clarity)
- Stripe (trust, premium B2B feel)
- Adapt for Vietnamese SME users — not overly minimal/cold; add warmth through micro-interactions and friendly empty states

## 5. Typography

Vietnamese characters require careful font selection (tonal diacritics: ắ, ề, ộ, ử, etc.).

### Recommended Font Stack
| Role | Font | Weights | Rationale |
|---|---|---|---|
| **Heading** | Be Vietnam Pro | 700, 800 | Designed specifically for Vietnamese; geometric, modern, professional |
| **Body / UI** | Be Vietnam Pro | 400, 500, 600 | Consistent family; excellent Vietnamese glyph coverage |
| **Monospace** | Geist Mono | 400 | `tx_hash`, GS1 IDs, blockchain data |

Load via `next/font` (web) and `expo-google-fonts` (mobile). Be Vietnam Pro is on Google Fonts.

### Scale (Tailwind)
- H1 marketing: `text-5xl lg:text-7xl font-extrabold`
- H2 section: `text-3xl font-bold`
- H3 card: `text-xl font-semibold`
- Body: `text-base` (16px) — never go below 14px for Vietnamese readability
- Caption: `text-sm text-slate-500`
- Mono: `font-mono text-sm` (blockchain data)

## 6. Website — Screen Inventory

The website has two zones: **Marketing** (public, for B2B prospects and general visitors) and **B2C Trace** (public, for end consumers who scanned a QR code).

### 6.1 Marketing Site

#### Landing Page (`/`)
Must feel like an **engaging discovery experience** — scroll through sections that tell the Vierify story. Sections in order:

1. **Hero** — Full-viewport. Headline + sub-headline + primary CTA ("Bắt đầu miễn phí" / "Start Free") + secondary CTA ("Xem demo"). Background: subtle animated gradient or a clean product screenshot. Vietnamese-first copy, language toggle in nav.

2. **Trust bar** — logos of certification/partner orgs or "Được tin dùng bởi..." (if any). If no real logos yet, use feature badges: "Polygon Blockchain", "GS1 Vietnam", "ISO trace-ready".

3. **Problem → Solution** — 2-column: left "Vấn đề" (supply chain opacity, food fraud), right "Giải pháp" (Vierify trace). Animated diagram of a supply chain resolving to a timeline.

4. **How It Works** — 3-step: Scan batch → Link chain → Consumer scans QR. Each step has an icon, short headline, 1-sentence body. Use scroll-triggered entrance animation.

5. **Feature highlights** — Cards for: GS1 Barcode scanning, Blockchain proof, Real-time genealogy, Document upload, B2C QR timeline. Each card: icon + title + 2-line description.

6. **B2C demo preview** — Show a mockup of the `/trace/[id]` page inside a browser frame. Clickable → opens a demo trace URL.

7. **Pricing** — Full pricing table (see Section 9 below). This is a required section.

8. **CTA banner** — "Bắt đầu truy xuất nguồn gốc ngay hôm nay" + sign-up button.

9. **Footer** — Links: Về chúng tôi, Tính năng, Bảng giá, Liên hệ, Chính sách bảo mật. Social links. Language toggle. Light/dark toggle.

#### Language Toggle
- Default: Vietnamese (`vi`)
- English (`en`) option in nav bar (flag icon + "EN" label)
- Persist to `localStorage`; use `next-intl` or simple context — keep it simple
- All marketing copy must have `vi` + `en` strings

### 6.2 B2C Trace Timeline (`/trace/[id]`)

Existing skeleton from T18 — Claude Design implements full visual treatment.

**Layout:** Single-page, vertical scroll. No nav sidebar. Mobile-first.

**Sections:**
1. **Header bar** — Vierify logo + "Truy xuất nguồn gốc" label. No full nav (consumer-facing).
2. **Batch identity** — Name, GS1 trace ID (monospace), source node name.
3. **Blockchain status badge** — Prominent pill: emerald "Đã xác minh trên Polygon" or amber "Đang xử lý blockchain". If confirmed: show `tx_hash` (truncated, copy button) + "Xem giao dịch trên Polygonscan Amoy" link.
4. **Stats row** — Quantity + UOM, Node type, Scan count. 3-column grid.
5. **Supply chain timeline** — Vertical timeline if parent batches exist (from genealogy API). Each node: batch name, date, node name (PII masked if `is_individual=true`). Animate on scroll.
6. **Document** — If `doc_hash` present: "Tài liệu đính kèm" section with icon + hash (truncated).
7. **Footer** — "Powered by Vierify" + link to landing page.

## 7. Mobile App — Screen Inventory

**Navigation structure (expo-router v6 groups):**

```

  (auth)/
    login.tsx          # Email + password login
  (app)/
    _layout.tsx        # Tab bar: Batches | Scan | Profile
    (batches)/
      index.tsx        # Batch history list

      new.tsx          # Create new batch
    (scan)/
      index.tsx        # QR scanner screen
    (profile)/
      index.tsx        # Profile + KYB status
```

### Screen Details

#### Login (`(auth)/login.tsx`)
- Vierify logo + tagline
- Email + password fields (large touch targets — factory floor use)
- "Đăng nhập" primary button
- Error state: KYB not approved → show "Tài khoản chờ phê duyệt" banner with support contact

#### Batch History (`(batches)/index.tsx`)
- Header: "Lô hàng của tôi" + create button (top right)
- List: each item shows batch name, GS1 ID (truncated), quantity + UOM, `bc_status` badge (pending/confirmed), date
- Pull-to-refresh
- Empty state: illustration + "Chưa có lô hàng nào. Tạo lô hàng đầu tiên."
- Skeleton loading state while fetching

#### Batch Detail (`(batches)/[id].tsx`)
- Batch name (large heading)
- **Blockchain status badge** — same emerald/amber system as web
- Stats: quantity, UOM, node, scan count
- `tx_hash` section (if confirmed): truncated hash, copy button, "View on Polygonscan" link (opens in-app browser)
- Document section: upload button (triggers `POST /batches/:id/document`) + current doc hash if set
- Parent batches section: list of linked parent batches with names
- QR code button: "Xem mã QR" → opens QR modal with the PNG data URL from `GET /batches/:id/qr`

#### Create Batch (`(batches)/new.tsx`)
- Form: Name, Quantity, Unit of Measure (picker), GPS (auto-fill from device), GS1 Trace ID (either scan or manual)
- Link parent batches: searchable list of own batches
- Submit → creates batch + shows success animation

#### QR Scanner (`(scan)/index.tsx`)
- Camera view with scanning frame overlay
- Success: animate frame → green + haptic feedback
- Scanned QR (a `/trace/{gs1_trace_id}` URL) → open in-app WebView showing B2C timeline

#### Profile (`(profile)/index.tsx`)
- Node name, KYB status badge
- Logout button
- App version

## 8. Animation Guidelines

**Mobile (reanimated v4 — STRICT):**
- Use ONLY: `useSharedValue`, `useAnimatedStyle`, `withTiming`, `withSpring`, `withSequence`, `runOnJS`
- Do NOT use: `Animated.createAnimatedComponent`, `useAnimatedValue`, or any v3 patterns
- Scan success: scale pulse (`withSpring`) on the scan frame
- Batch list item entrance: `withTiming` opacity + translateY on mount
- Badge state change: `withTiming` color transition via interpolateColor

**Web:**
- Use Tailwind `transition-*` utilities for hover/focus states
- Use `framer-motion` for scroll-triggered section entrances (landing page only)
- Keep animations under 300ms for UI interactions; 500ms for page-level entrances
- Respect `prefers-reduced-motion` — wrap framer-motion in a motion-safe check

**General rules:**
- Animations must signal state, not decorate
- Never block interaction during animation
- Loading skeletons (not spinners) for list/data fetches

## 12. Technical Constraints (Hard Rules)

1. **Web is Server Components by default** — add `"use client"` only for interactive components (forms, modals, framer-motion wrappers)
2. **Mobile NativeWind v4** — `className` prop only; import `global.css` at the top of `_layout.tsx`
3. **Mobile reanimated v4 only** — no v3 patterns (see Section 8)
4. **No new API routes** — all data is available via existing tRPC procedures and REST endpoints (see Section 13)
5. **PII anonymisation** — `is_individual=true` nodes show `"***"` for name and address on the B2C trace page. Do not display raw data
6. **`audit_log` is append-only** — never add delete or update UI for audit entries
7. **Supabase anon RLS** — the B2C trace page reads using the anon key; RLS must be configured (done separately by Codex) — do not add server-side auth to the public trace page

## 13. API Surface for Wiring

### tRPC (authenticated — B2B mobile)
| Procedure | What it returns |
|---|---|
| `batches.list({ nodeId, limit })` | Array of trace batches for the logged-in node |
| `batches.get({ id })` | Single batch with `bc_status`, `tx_hash`, `doc_hash` |
| `batches.create(input)` | Creates batch + queues blockchain job |
| `batches.getByTraceId({ gs1TraceId })` | Public — used for B2C trace page |
| `genealogy` (via REST) | See REST below |

### REST endpoints
| Endpoint | Purpose |
|---|---|
| `GET /batches/:id/qr` | Returns `{ qrDataUrl, traceUrl, gs1TraceId }` — use `qrDataUrl` to render QR image |
| `POST /batches/:id/document` | Multipart — uploads file, returns `{ signedUrl, docHash }` |
| `POST /batches/:child_id/parents` | Links parent batches to child |
| `PATCH /admin/nodes/:id/kyb` | Admin only — updates KYB status |

### Supabase direct (B2C web, anon key)
| Query | Returns |
|---|---|
| `trace_batch.select("*, supply_chain_node(*)").eq("gs1_trace_id", id)` | Full batch + node; PII must be masked in `data.ts` before render |
