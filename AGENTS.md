## RBAC — Apply to Every New Route

tRPC procedure tiers (defined in `trpc.ts` — do not redefine):

| Procedure | Who can call |
|---|---|
| `publicProcedure` | Anyone — B2C endpoints only |
| `readProcedure` | admin, merchant, viewer |
| `merchantProcedure` | admin, merchant |
| `adminProcedure` | admin only |

Multi-tenant scoping — **mandatory** for all read/write queries by non-admins:

```typescript
import { getTenantOrgId } from "../../context";

// Router layer — pass orgId through:
list: readProcedure
  .input(listSchema)
  .query(({ ctx, input }) => listItems(input, getTenantOrgId(ctx.user)))

// Service layer — filter by orgId when set (undefined = admin, sees all):
export async function listItems(input, orgId?: string) {
  if (orgId) {
    // JOIN with supply_chain_node WHERE orgId matches
  }
  // else: no filter — admin path
}
```

## Critical Patterns

### tRPC procedure

```typescript
export const batchesRouter = router({
  create: merchantProcedure
    .input(createBatchSchema)
    .mutation(({ ctx, input }) =>
      createBatch(input, ctx.user.id, getTenantOrgId(ctx.user))
    )
});
```

### BullMQ worker

```typescript
const worker = new Worker("blockchain", processor, {
  connection,
  defaultJobOptions: {
    attempts: 5,
    backoff: { type: "exponential", delay: 2000 }
  }
});
// Always configure dead-letter queue for failed jobs
```

### Supabase Auth in Fastify (REST routes)

```typescript
const user = await requireAdminUser(request.headers.authorization);
// throws UNAUTHORIZED or FORBIDDEN automatically
```

### PII anonymisation (B2C responses)

```typescript
function anonymiseNode(node: NodeProjection): NodeProjection {
  if (!node.isIndividual) return node;
  return { ...node, name: "***", nodeAddress: "***" };
}
```

### Drizzle migration workflow

```bash
# After changing apps/api/src/db/schema.ts:
pnpm --filter @vierify/api drizzle-kit generate
pnpm --filter @vierify/api drizzle-kit migrate
# Commit BOTH schema.ts AND the generated migration file in the same PR
```

## Do NOT

- Change business rules (PLAN.md § Business Rules) without Claude's sign-off
- Use `publicProcedure` for any mutation or user-scoped read
- Skip `getTenantOrgId()` for merchant/read queries — cross-org leak is a security bug
- Block API responses on Polygon transactions — always async via BullMQ
- Store PII in any field sent to blockchain
- Delete or UPDATE `audit_log` — append-only
- Use `any` in TypeScript — use `unknown` and narrow
- Skip Zod validation on tRPC inputs
- Hard-code secrets
- Modify `apps/web/` or `apps/mobile/` UI — those belong to Claude
