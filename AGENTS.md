## Critical Patterns

### tRPC procedure (API)
```typescript
// Always use Zod for input validation
export const batchesRouter = router({
  create: protectedProcedure
    .input(z.object({ name: z.string(), quantity: z.number().positive(), ... }))
    .mutation(async ({ input, ctx }) => {
      // 1. validate business rules (Mass Balance etc.)
      // 2. write to PostgreSQL via Drizzle
      // 3. enqueue blockchain job (do NOT await)
      await blockchainQueue.add('hash-batch', { batchId: result.id })
      return result  // return immediately
    }),
})
```

### BullMQ worker (blockchain)
```typescript
// Worker runs in same Render process, separate from API handler
worker.process(async (job) => {
  const batch = await db.select().from(traceBatch).where(eq(traceBatch.id, job.data.batchId))
  const hash = hashBatch(batch)  // SHA-256, PII stripped
  const tx = await contract.writeHash(batch.id, hash)
  await tx.wait()
  await db.update(traceBatch).set({ txHash: tx.hash, bcStatus: 1 }).where(...)
})
// Always configure: attempts: 5, backoff: { type: 'exponential', delay: 2000 }
// Always add a dead-letter queue for failed jobs
```

### Supabase Auth in Fastify
```typescript
// Verify JWT from Authorization: Bearer <token> header
const { data: { user }, error } = await supabase.auth.getUser(token)
if (error || !user) throw new TRPCError({ code: 'UNAUTHORIZED' })
ctx.userId = user.id
```

### PII anonymisation (B2C responses)
```typescript
// Any query returning node data for B2C must apply this
function anonymiseNode(node: Node) {
  if (!node.isIndividual) return node
  return { ...node, name: '***', nodeAddress: '***' }
}
```

### Drizzle migration workflow
```bash
# After changing schema.ts:
pnpm --filter api drizzle-kit generate
pnpm --filter api drizzle-kit migrate   # runs against DATABASE_URL
# Commit both schema.ts and the migration file
```

## Do NOT

- Change business rules (PLAN.md § Business Rules) without Claude's sign-off
- Block API responses on Polygon transactions — always async via BullMQ
- Store PII in `doc_hash` or any field sent to blockchain
- Delete from `audit_log` table — it is append-only
- Use `any` in TypeScript — use `unknown` and narrow
- Skip Zod validation on tRPC inputs
- Hard-code secrets — use environment variables
