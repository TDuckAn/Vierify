# /pipeline — Autonomous Claude↔Codex Pipeline

Run one iteration of the autonomous pipeline. Each call advances the pipeline by exactly one phase action, then reschedules itself via ScheduleWakeup.

## How to Start
```
/pipeline start "your sprint goal here"
```
This initialises PIPELINE.md and enters the loop automatically.

## Loop Logic

Read `PIPELINE.md` first (< 40 lines). Then act based on `phase`:

---

### phase: planning
1. Read current sprint tasks from PLAN.md that are `☐` and owned by Codex
2. Write/update `AGENTS.md` sprint section with task specs (acceptance criteria, patterns)
3. Write PIPELINE.md with first pending Codex task, set `phase: codex`, `codex_invoked: false`
4. ScheduleWakeup 10s (proceed immediately to codex phase)

---

### phase: codex
**If `codex_invoked: false`:**
1. Generate `PIPELINE_TASK.md` in repo root (< 25 lines, see format below)
2. Run in background: `codex --approval-mode full-auto -q "$(cat PIPELINE_TASK.md)"`
3. Update PIPELINE.md: `codex_invoked: true`, `updated: <now>`
4. ScheduleWakeup 90s (give Codex time to work)

**If `codex_invoked: true`:**
1. Run: `git log --oneline -5`
2. If any commit message contains `[codex-done]` after the `updated` timestamp → update PIPELINE.md `phase: review`, ScheduleWakeup 10s
3. Else → ScheduleWakeup 60s (still waiting)

---

### phase: review
1. Run `git diff --stat HEAD~1` — note which files changed
2. Read ONLY the changed files (not full codebase)
3. Run `pnpm typecheck 2>&1 | tail -20`
4. **If typecheck passes:**
   - Check if there are `☐` Claude tasks for this sprint item in PLAN.md
   - If yes → set `phase: claude`, ScheduleWakeup 10s
   - If no → set `phase: ci`, ScheduleWakeup 10s
5. **If typecheck fails:**
   - Write the error summary (≤ 60 words) to PIPELINE.md `handoff` field
   - Set `codex_invoked: false`, `phase: codex`
   - ScheduleWakeup 10s (Codex will re-run with error context)

---

### phase: claude
1. Read PIPELINE.md `handoff` for context on what Claude needs to do
2. Implement Claude's tasks (UI components, Playwright tests, PLAN.md updates)
3. Commit with conventional commit message
4. Update PIPELINE.md: `phase: ci`, `updated: <now>`
5. ScheduleWakeup 30s (wait for GitHub to register the push)

---

### phase: ci
1. Run: `gh run list --limit 1 --json status,conclusion,databaseId,headSha`
2. **If status=completed AND conclusion=success:**
   - Mark current task `✅` in PLAN.md
   - Find next `☐` task in current sprint owned by Codex or Claude
   - If found → update PIPELINE.md with next task, reset `codex_invoked: false`, set appropriate phase
   - If none → set `phase: done`, ScheduleWakeup 10s
3. **If status=completed AND conclusion=failure:**
   - Run: `gh run view <databaseId> --log-failed 2>&1 | tail -40`
   - Fix the failure (edit files, commit)
   - ScheduleWakeup 60s (wait for new CI run to start)
4. **If status=in_progress or queued:**
   - ScheduleWakeup 120s

---

### phase: done
1. Update PLAN.md: mark sprint complete, add summary line
2. Commit: `chore: Sprint ## complete — all tasks ✅`
3. Output to user: "Pipeline complete. Goal achieved: [goal]. Tasks completed: [list]."
4. Do NOT call ScheduleWakeup — loop ends here.

---

### phase: error
1. Output the error to user clearly
2. Do NOT call ScheduleWakeup — user must intervene
3. User can resume by editing PIPELINE.md to fix the error and running `/pipeline` again

---

## PIPELINE_TASK.md Format (generated by Claude, read by Codex)

```markdown
# Task: T##
**Goal:** [one sentence — what to implement]
**Acceptance criteria:**
- [bullet 1]
- [bullet 2]
- [bullet 3]
**Key patterns:** See AGENTS.md §[Module Conventions] and §[Critical Patterns]
**Files to create/modify:** [list only affected files in apps/api/]
**Done signal:** Final commit message MUST include `[codex-done]`
**Must NOT touch:** apps/web/, apps/mobile/, e2e/, *.spec.ts, *.test.ts
```

---

## Token Efficiency Rules
- Never re-read PLAN.md in full each turn — read only the sprint table section
- Never re-read AGENTS.md in full — read only the relevant module section
- For `git diff`, use `--stat` first, then read only files that actually changed
- Keep PIPELINE.md `handoff` field ≤ 80 words
- CI log: read only `--log-failed`, last 40 lines max

---

## User Interruption
If the user sends a message mid-loop, Claude reads it at the next ScheduleWakeup and decides:
- Minor note → acknowledge, continue loop
- Direction change → update PIPELINE.md accordingly, continue
- "stop" / "pause" → set `phase: idle`, stop ScheduleWakeup
